## 1. 목표 (What)

30명 파일럿 운영 시 **개별 사용자 과소비·전체 예산 초과·카운터 경합**을 방지하는 **쿼터 정책의 상세 스펙**을 확정한다. 본 플랜은 **정책·UX·구현 의사코드까지**만 결정하고, 실제 서버 코드는 PLAN-028(API 프록시)에서 구현.

## 2. 배경·동기 (Why)

- `PLAN-026` Supabase 채택 → `roadmap.md` §3 A.1에서 쿼터 카운터 원자적 UPDATE 요구
- `roadmap.md` §6.1: 리셋 주기·초과 UX·원자성·투명성·예산 상한 — 5개 결정 지점 열거, 아직 미확정
- §7 "MCP·다수 절감 수단 적용 후에도 월 $50 하드 상한 필요" — 스케일 아웃 대신 쿼터가 1차 방어선
- 30명에게 공개한 쿼터 정책을 **번복하면 신뢰 하락** (§9 리스크) → P1 시작 전 반드시 확정

## 4. 설계 결정 (확정 필요 항목)

### 4.1 리셋 주기 — 2안 비교
| 안 | 설명 | 장점 | 단점 |
|---|---|---|---|
| A | **월초 일괄** (매월 1일 00:00 UTC) | 구현 단순, 크론 1회 | 말일 가입자 손해 |
| B | **가입일 rolling** (가입 D+30 기준) | 공정 | 사용자별 reset_at 관리 복잡 |

→ 권고 후보: **A (월초 일괄)** — 파일럿 30명 규모엔 공정성 손실 미미, 구현·디버깅 단순

### 4.2 초과 UX — 3안 비교
| 안 | 설명 | 사용자 경험 | 비용 통제 |
|---|---|---|---|
| A | **하드 차단** ("이번 달 쿼터 소진") | 불만 ↑ | 강함 |
| B | **경고 + 대기** (다음 달 자동 재개) | 중립 | 강함 |
| C | **긴급 추가 (+$)** | 유연 | 약함 (Phase C에 적합) |

→ 권고 후보: **B** — 파일럿 기간 친절성 유지, 월초 자동 재개 문구로 안심

### 4.3 원자성 — 구현 패턴
```sql
-- 체크·차감을 한 트랜잭션으로
UPDATE quotas
SET used_count = used_count + 1
WHERE user_id = $1
  AND used_count < limit_count
  AND period_start > now() - interval '1 month'
RETURNING used_count, limit_count;
-- 0 rows returned → 쿼터 초과 (차감 없음, 안전)
-- 1 row returned → 통과, 이 값으로 API 호출 진행
```

- Edge Function에서 위 쿼리 실행 → 결과로 분기
- API 호출 실패 시 별도 `UPDATE used_count = used_count - 1` 롤백 (보상 트랜잭션)

### 4.4 투명성 UI
- Header 우측에 "API X/10 이번 달" 표시
- 80% 초과 시 노랑, 100% 빨강
- 클릭 시 모달 — 최근 호출 5건·리셋일 표시
- 갱신: API 호출 성공·실패 직후 즉시 (낙관적 UI)

### 4.5 전체 예산 상한
- **하드 상한**: 월 $50 (`api_usage_log`에서 SUM(cost_usd) 계산)
- 도달 시: 새 API 호출 차단, 모든 사용자에게 공지 배너
- Edge Function 내 체크 로직: 개인 쿼터 → 전체 예산 → 호출 순서
- 80% 도달 시 운영자(본인)에게 email 알림 (Resend)

### 4.6 어뷰즈 방지 — 파일럿 단계
- 계정당 정식 도메인 email 필수? (아니오, 파일럿은 gmail OK)
- IP rate limit (Edge Function에서 분당 10req) — Supabase 내장 기능 활용
- 동일 이메일 복수 계정? → magic link 자체 차단

### 4.7 기본 쿼터 값
| 역할 | 월 API 호출 한도 | 근거 |
|---|:-:|---|
| viewer (30명 파일럿) | **10회** | 30명 × 10회 × $0.16/call = $48 < $50 상한 |
| editor | 30회 | 편집자 필요량, 3명 가정 시 추가 $15 |
| admin (본인) | **무제한** | 개발·테스트 필요. 단 $50 예산 공유 (본인도 카운트) |
